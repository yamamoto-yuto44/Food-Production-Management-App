# 食品製造管理アプリ

## 概要
食品製造現場の実務を想定した在庫・生産管理アプリケーションです。

当日の生産登録を行うと、レシピに基づき、原料在庫が自動で減少し、生産履歴・在庫履歴として記録されます。

また、原料発注を行うことで、入荷処理によって在庫を補充することができます。

ユーザー権限（USER / ADMIN）による機能制御も実装しています。

## 主な機能

### ① 生産登録

製品・製造日・数量を入力して生産登録を行います。  
登録時にはレシピを参照し、使用原料量に基づいて原料在庫が自動で減少します。  
また、在庫変動は在庫履歴として記録されます。

![production-create](images/production-create.png)

---

### ② 生産履歴管理

登録された生産記録を一覧で確認できます。  
製造日順（新しい順）に並び替えて表示しています。

![production-list](images/production-list.png)

---

### ③ 原料在庫管理

原料ごとの現在在庫数を一覧で確認できます。  
在庫不足時には発注を行うことができます。

![material-list](images/material-list.png)

---

### ④ 在庫履歴確認

原料の在庫増減履歴を確認できます。  
生産による在庫減少や、入荷による在庫増加を履歴として記録しています。

![material-stock-history](images/material-stock-history.png)

---

### ⑤ 発注管理（管理者機能）

管理者ユーザーは原料の発注登録と入荷処理を行うことができます。  
入荷処理を行うことで在庫が自動で更新されます。

![orders](images/orders.png)

## 設計上の工夫

### Service層でビジネスロジックを管理
生産登録時の在庫減少処理や発注入荷による在庫増加処理などの
ビジネスロジックはService層に集約しています。

これによりControllerはリクエスト処理のみを担当し、
責務分離を意識した設計としました。


### レシピを参照した在庫更新処理
生産登録時にはレシピテーブルを参照し、
製品ごとに設定された使用原料量をもとに在庫を自動で減少させる処理を実装しています。


### 在庫履歴テーブルによる履歴管理
原料在庫の増減履歴を material_stock_history テーブルで管理しています。
生産による在庫減少や入荷による在庫増加を履歴として記録できるようにしました。


### DTOによるデータ受け渡し
一部の画面ではDTOを使用し、ControllerとView間のデータ受け渡しを行っています。


### Spring Securityによる認証・認可
Spring Securityを利用してログイン認証を実装しています。
また、USER / ADMIN のロールによるアクセス制御を行い、
SecurityConfigでURLごとの権限制御を行っています。

## 今後の改善点

### ページネーションの実装
生産履歴や在庫履歴はデータ件数が増えると
画面表示や読み込み速度に影響する可能性があります。

そのため、Spring Data JPAの `Pageable` を利用し、
ページネーション機能を実装することで
大量データでも見やすく扱えるよう改善したいと考えています。


### テストコードの追加
現在は手動での動作確認を中心に開発を行っていますが、
今後はJUnitを利用してService層のテストコードを追加し、
在庫減少処理や入荷処理などのビジネスロジックの
動作保証を行えるようにしたいと考えています。


### 入力バリデーションの強化
現在も基本的な入力チェックは行っていますが、
数量の異常値や在庫不足時のエラーハンドリングなど、
より安全にデータを扱えるようバリデーションを強化したいと考えています。


### UI/UXの改善
現在、Bootstrapを使用した基本的なUIを実装していますが、
ユーザーが直感的に操作できるように操作性の向上を検討しています。
